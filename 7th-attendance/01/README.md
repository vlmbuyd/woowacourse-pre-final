## 기능 요구사항

### 입력

- [x] 요구사항을 입력받는다. (1,2,3,4,Q)
- [ ] 입력받은 요구사항에 따라 분기하여 처리한다. (e.g. 1번입력 시 "출석확인" 처리)
  - [ ] "출석 확인" 입력 시 닉네임과 등교 시간을 입력받는다.
  - [ ] "출석 수정" 입력 시 닉네임과 수정 날짜, 변경 시간을 입력받는다.
  - [ ] "크루별 출석 기록 확인" 입력 시 닉네임을 입력받는다.
- [ ] 사용자가 종료할 때까지 종료되지 않으며, 해당하는 기능 수행 후 초기 입력 프롬프트를 다시 띄운다.
- [x] [예외] 잘못된 값을 입력할 경우 `ERROR` 메시지와 함께 애플리케이션 종료한다.

### 기능

**csv 파싱**

- [ ] csv 파일을 불러온다.
- [ ] 리스트 형태로 파싱한다.
- [ ] 정적 데이터로 저장한다.

**데이터 관리**

- [ ] 교육 시간 바탕으로 출석 데이터의 출석/지각/결석을 판단하여 플래그를 추가한다.
- [ ] 크루별 출석/지각/결석 기록을 필터링한다.
  - [ ] 기록을 바탕으로 경고/면담/제적 대상자인지를 판단한다.

**출석 확인**

- 입력된 내용을 바탕으로 안내사항을 출력한다.
  - [ ] 주말 및 공휴일에는 출석을 받지 않는다.
  - [ ] 오늘 날짜에 출석했다면 데이터를 추가한다.
    - [ ] 출석 기록을 출력한다
  - [ ] 이미 출석한 경우, 출석 할 수 없다.
    - [ ] 수정 기능을 이용하도록 안내 문구를 출력한다.

**출석 수정**

- [ ] 수정 기록을 출력한다.

**크루별 출석 기록 확인**

- [ ] 입력받은 크루의 전날까지의 출석 기록을 출력한다.
  - [ ] 출석/지각/결석 총 횟수를 출력한다.
  - [ ] 경고/면담/제적 대상자일 경우 안내 문구를 출력한다.

**제적 위험자 확인**

- [ ] "제적 위험자 확인" 입력 시 멤버 별 조회 결과를 출력한다.
  - [ ] 제적 대상자 -> 면담 대상자 -> 경고 대상자순으로 출력한다.
  - [ ] 같은 항목 내에서는 지각을 결석으로 환산하여 내림차순 정렬한다.
  - [ ] 출석 상태가 같으면 닉네임으로 오름차순 정렬한다.

**종료**

- [ ] Q 입력 시 애플리케이션을 종료한다.
